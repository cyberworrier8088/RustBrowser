Mini Browser Book

Welcome to the Mini Browser documentation. This page explains the HTML tags, browser features, and internal architecture currently supported by Mini Browser.

Headings (h1 - h6)

Headings are used for titles and section names. Larger heading numbers create smaller text. 

<h1>Main Title</h1>
<h2>Section Title</h2>
<h3>Sub Section</h3>
<h4>Heading</h4>
<h5>Heading</h5>
<h6>Heading</h6>

Paragraph (p)

Paragraphs are used for normal text content. 

<p>Hello World!</p>

Links (a)

Links allow navigation between pages and websites. 

<a href="https://example.com">
Visit Example
</a>

Images (img)

Images display pictures from local files or internet URLs. 

<img src="cat.png">

Lists (ul, li)

Lists are used for displaying multiple items. 

<ul>
    <li>Rust</li>
    <li>Browser</li>
    <li>HTML</li>
</ul>

Tables

Tables display information using rows and columns. 

<table>
    <tr>
        <td>Name</td>
        <td>Age</td>
    </tr>

    <tr>
        <td>Nabhan</td>
        <td>15</td>
    </tr>
</table>

Div

Div elements are containers used to group content together. Most modern website layouts are built using div elements. 

<div>
    <h3>Hello</h3>
    <p>Inside Div</p>
</div>

Span

Span is an inline container used to style or group text. 

<span>Hello Span</span>

Supported Browser Features

  • HTML Parsing
  • DOM Tree
  • Layout Tree
  • Rendering Engine
  • Image Loading
  • Downloads
  • Tabs
  • Bookmarks
  • History Navigation
  • Text Selection
  • Copy To Clipboard
  • Basic CSS Colors
  • Basic Font Sizes

Browser Architecture

Mini Browser uses a simplified browser architecture inspired by real browsers. 

HTML
 |
DOM Parser
 |
DOM Tree
 |
Layout Tree
 |
Renderer
 |
Pixels

The DOM Tree stores document structure, the Layout Tree calculates positions and sizes, and the Renderer converts everything into pixels displayed on screen. :)
